Task 1 - Drawing Single-Color Triangles

First, we had to implement the rasterize_triangles() function where we were given three vertices that represented the points of the triangles and a Color object to fill the triangle with. We first created a bounding box containing just the triangle we were given. This optimized our code by giving us a box of points that were closest to to the given triangle. We then iterated through each point in the bounding box and conducted a line test. The line test consisted of taking the dot product between a point/vector and a line on the triangle. This could return three values:

“-”: the point is below the line of the triangle

“=”: the point is on the line of the triangle

“+”: the point is above the line of the triangle

For each point we iterated through, the line test was conducted three times for each line of the triangle. If the values returned for all three line tests were either “+++” or “—-”, this means that point is inside the triangle, and we would then fill the point with our given color.

Task 2 - Antialiasing by Super-sampling

In our supersampling algorithm, we split each individual pixel into sampling rate number of pixels by resizing our sample_buffer each time the either the sampling rate or the screen size was changed with width * height * rate. This allowed us to run the Three Line Tests on more pixels and downsample the pixels’ colors (take the average of the supersampled pixels within the original pixel).

Supersampling is useful because it adjusts the resolution of an image to the screen, while applying anti-aliasing. Supersampling removes aliasing by averaging out the high frequencies and then sampling at a lower resolution/frequency while maintaining the same quality of a high resolution/frequency image.

The modifications we made to the rasterization pipeline were: creating a quadruple for loop instead of a double for loop to account for the iteration of supersampled pixels, indexing by a factor of the square root of the sampling rate sample_buffer[y * (width * sqSr) + x] instead of just the width to properly traverse columns and rows of the array, assigning the sampling rate number of pixels a color in fill_pixel() instead of just 1, and calculating the appropriate offset (distance of the super sampled center in relation to the corner of the original pixel).

Task 4 - Barycentric Coordinates

Barycentric coordinates describe the position of a coordinate in relation to the 3 vertices of a triangle. The coefficients are like the strength that the vertex has on the coordinate, or how much the vertex pulls on the coordinate.

Task 5 - “Pixel Sampling” for Texture Mapping

Pixel sampling is the process of mapping a coordinate in screen space (x, y) to its corresponding coordinate in texture space (u, v). In order to perform texture mapping we used pixel sampling to map the coordinate and get its color value, along with the pixel’s neighboring color values.

Nearest-neighbor and bilinear sampling were the two different pixel sampling methods that we used. Nearest-neighbor takes (u, v) and rounds the values to the nearest integer, which is used in the get_texel() call. Bilinear is similar, as we round (u, v) to the nearest integer, but we also find its 4 neighbors by adding an offset of 0.5f. Using these 4 neighbors, we interpolate between their color values based on the position of (u, v).

The nearest sample images are slightly brighter than the bilinear sample images. There are also more obvious jaggies in the nearest sample images compared to the bilinear sample images for both sampling rates.

The difference between nearest-neighbor and bilinear sampling is that bilinear sampling generally produces smoother images. This is because we interpolate between color values instead of having a binary mapping (essentially using another form of averaging). There will be a large difference between the two methods when there is an image where the colors at neighboring pixels vary greatly. Following the logic from above, if (u, v) had RGB neighbors, nearest-neighbor would assign (x, y) either red, green, or blue depending on which pixel it is closest to. Meanwhile, bilinear sampling would interpolate between the 3 colors and produce a color somewhere with the SVG RGB triangle example in Task 4.
